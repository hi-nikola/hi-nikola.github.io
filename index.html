谈谈我对摄影的一些思考

2022-04-30

我对摄影的思考，我对拍照的看法。

2020年疫情被封在家里几个月，实在无聊，就玩起了买了一年多的相机，首先是是对相机的了解，其次是对摄影技术的了解（光圈、快门、ISO、对焦等），然后是构图、光线，最后是审美的提升。现在发现，学习摄影最直接的方式是看说明书+看摄影作品。

我喜欢的摄影师（比较有代表性的）：
川内伦子
埃里克·索斯
威廉·艾格斯顿
214

并且在这里我强烈建议，如果可以的话，去看看你喜欢的摄影师的个人摄影网站（经济条件允许，可以选择画册）。这里我推荐原画册的这篇公众号文章：为什么要看摄影师的个人网站？

画册比较贵、展览不常有、网络信息杂，其实个人网站是个性价比很高的欣赏窗口。
某种意义上来说，散见于互联网的作品是很难get到真正魅力的，在个人网站上，加持设计感、交互感和作品系统性，才能将摄影的魅力真正展现。
图虫和500px等摄影门户网站，无论再独立，也终究是为流量和资本服务，所见推荐依然是投大众所好，底层逻辑依然以单张的视觉猎奇为主。
而通过个人网站可以深入了解摄影师创作的脉络，看到他们的项目、成长过程，还有文字、视频材料予以辅助。同时，可以学习他们的个人品牌运营方式。

同时，也给出了推荐：水谷吉法、细仓真弓、井津建郞、杉本博司 等

其实最早的想法是搭建一个个人摄影网站的，但是考虑到一旦放入大量图片，就会影响网页的响应速度（甚至图片加载都要花费很久的时间），与其如此不如把自己的摄影集放在门户网站上，如果你喜欢摄影同时也想看看我都拍了些什么，可以从我的联系方式找到我的摄影作品。

关于添加评论功能遇到的小问题

2022-04-29

一、缺少的评论区

昨天搭建好博客后，迫不及待的更新了一篇文章，十分的激动，虽然是胡乱凑了几个字，好歹也清楚说明了自己搭建博客的原因，但是总觉得还缺少了什么，哦，对！就是评论区（留言板），我觉得增加评论区是更加方便交流的，也会让其他访问网站的用户更容易留言（如果没有评论区，可能需要先找到我的个人邮箱，然后打开自己的邮箱输入内容再发送给我，这个无疑增加了沟通的成本）

二、尝试评论功能-twikoo

说干就干，首先我在hexo交流群发现了一个非常不错的博客：Twikoo ，可以使用QQ登录评论，能够自动补全昵称和邮箱，是不是很厉害！

结果今天从起床开始折腾到现在，还是失败了，原因是我采用的的这个主题没有配置twikoo，自己配置是需要掌握CDN的，我从网上搜了一下相关资料，觉得对我来说有点困难于是作罢。

三、新的希望-giscus

giscus是由 GitHub Discussions 驱动的评论系统。通过看官方文档giscus顺利的解决了问题（期间，也出现了一些小问题，比如在giscus官网输入我的仓库地址时，怎么输入也不正确，后来才发现不能直接copygithub左上角的地址；又比如配置yml文件时，” : “后面一定要跟个空格再输入内容，否则会导致配置失败）

解决了这些小问题，评论功能终于成功了，可以点赞可以发送火箭。

这个评论模块功能强大，但是有点点遗憾的就是只能够用github账号先登里再评论，不能使用其他账号。先不管了，等后面博客经验丰富了再配置吧。

四、总结感悟

经历一些波折终于还是完成了想要的功能。不禁感慨，看似容易的功能，要在自己的博客上实现还是要花一些功夫的。而且，这个过程中是需要不断地看文档解决问题，也真的要感谢神仙大佬们的文档了，真的太详细了！！能够让初学者按图索骥，摸清楚门路，这才是互联网的精神：开放、分享。

关于我

2022-04-28

为什么做个人博客主页？

有三点原因：
一、随着年龄的增长，我越来越发现“输出”的重要性，我想通过博客的记录来整理自己的思绪，把自己一些零碎的想法进行加工，能够完整呈现出来。
二、对各种媒体的内容太失望了。（抖音、头条、知乎、微博、豆瓣、贴吧等）企图制造各种流量话题（本人对实时热点也并不追捧），且这些内容质量不高， 因此不如把时间花在自我思考、阅读以及产出内容上。
三、近一个月互联网上，我看到的比较有深度的文章基本都是通过个人博客，有的文章看完后可以沉思很久，仿佛和博主进行了深度的对话，我相信博客连接着这个网络世界里相似的人，我也希望通过博客能够认识更多有趣的人！

最近也看到大佬的一篇博客：
为什么你应该（从现在开始就）写博客
这篇文章已经很清楚明确的说明了理由：我们所能做的最差的一个决策莫过于停止做一件没有任何坏处，却有一大堆好处的事情。
这里引用作者的观点进行补充：

  1. “教”是最好的“学”。如果一件事情你不能讲清楚，十有八九你还没有完全理解。绝大多数人应该都知道在程序员行业面试官经常要求你讲解一个东西给他听，他会说他不懂这个东西（他如果真的不懂的话效果其实是最好的），而你的任务则是说到让他理解为止。如果你选择经常总结自己的知识体系，并说出来给你的读者听，你就会发现你自己创造了这样的机会。
  2. 一开始的时候你是因为要写博客而去使劲地思考和总结，指望给出令人眼睛一亮的东西，到了后来，就变成了因为你习惯了思考和总结，因为你意识到书写是更好的思考，你就必须使你的想法成为文字。
  3. 无论如何，不用急于求成，在一个主题上深入下去思考，总能挖到别人挖不到的角落。你能让一个问题在大脑中停留的时间越长，就越是能够发现新的东西，一般来说，我认为有价值的问题我会让他在意识或潜意识中待短则一个星期，长则一个月（视问题大小而定），利用走路吃饭的时间琢磨（我发现很多我佩服的人也都有这个习惯），有时即便已经想通了写下来了发出去了，大脑仍然还是会在回味问题，还没有把它撤出潜意识，然后看到某篇文章或某本书的时候忽然又有所新的感悟。
    能够把问题长时间停靠在潜意识中是一种技能，能够带来很大的好处，停留得越长你越琢磨得透彻，比别人看到的就越多。

为什么选择 hexo+github

我尝试过多种搭建博客的方法，最终发现hexo静态页面简洁方便，只需要注册github和域名就可以直接使用，不需要云服务器（又可以省下一大笔），而aircloud主题是我在众多hexo主题中一眼看中，理由就是：简洁，简洁，还TM是简洁！完全满足了我对主题的需求。
